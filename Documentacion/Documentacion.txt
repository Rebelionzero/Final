Atributos: tabla, id, nombre, respuesta, conectar

Funcion del objeto: Identificar los distintos elementos a ser borrados de la base de datos y borrarlos con exito utilizando para ello a otros objetos.

Clases con las que interactua: Queries.php

Vistas con las que interactua: - Ninguna - 

Metodos:
*borrar_bd:
	-Diferenciar el tipo de tabla de la cual se va a realizar la eliminacion en la base de datos y:

	1) En el caso de que la tabla sea de marcas o categorias, se crea y ejecuta la query (mediante la clase Queries) que verifique que en la base de datos la marca o categoria a borrar no se encuentre siendo utilizada por un producto. Ademas de crear y ejecutar la query para borrar el producto de la base de datos. En ambos casos se genera un mensaje de respuesta que se envia al controlador correspondiente.

 ComprobarObra:
/**********************/

Controladores: controlador-obras.php

Atributos: titulo, descripcion, autor, anio, categoria, museo, imagen, mail, validaciones, errores

Clases con las que interactua: - Ninguna -

Vistas con las que interactua: - Ninguna -

Metodos: 

 Publicos:
 *verificar:
		- utiliza uno de los atributos/arrays de la clase: validaciones
		- llama al metodo privado setear_errores
		- la funcion de este metodo es almacenar en el array validaciones los resultados que arrojan los llamados a las funciones privadas, a las cuales se les pasa los parametros que el constructor almacena en los atributos.

		1) validaciones almacena los resultados de las comprobaciones segun el tipo de informacion que se busca verificar, ej: si la informacion que se le pasa contiene el limite de caracteres necesarios para su aprobacion.

		2) el llamado a setear_errores envia el contenido del array validaciones.

 Privados: 
	
  Todos los metodos privados a excepcion de setear_errores devuelven false si no existe ningun error de comprobación, si existe un error en la comprobacion se envia un mensaje de error.

 *vacio: se le envia un valor y este comprueba si dicho valor es un string vacio o no.

 *setear_errores:
		- se le envia el array validaciones con las comprobaciones hechas por los metodos privados dedicados a cada campo
		- el metodo verifica el array valor por valor, chequeando que dicho valor sea falso:

		1) si es falso significa que no hubo ningun error en la validacion, y que el dato ingresado esta dentro de los limites (largo de string, ausencia de caracteres extraños, etc).

		2) si no es falso, significa que hubo un error y dicho error se insaerta en el array errores, el cual se devuelve al front end y se muestra como mensaje de error al usuario.

 *validar_titulo:
		- comprueba el largo del string que corresponde al nombre de la obra, y lo pasa por una expresion regular la cual solo permite texto sin numeros ni espacios como nombre:

		1) Si es mayor o menor al largo requerido o si se ingresan caracteres extraños se devuelve un mensaje de error

		2) si el string ingresado cumple con los requisitos se devuelve false

 *validar_descripcion:
		- se utiliza la funcion trim para quitar espacios en blanco al inicio y al final del string, y se verifica que el largo del string no supere los 255 caracteres. De ser asi se envia un error, sino false.

 *validar_select:
		- comprueba que el valor del select enviado sea distinto a 'seleccione', si es igual a 'seleccione' significa que el cliente no eligió ningun autor, año, categoria y/o museo.

 *validar_img:
		- se valida que se haya subido un archivo a traves del valor del indice 'error' del array del archivo.
		- se verifica que la extension del archivo sea correcta (jpg, gif, png, jpeg, pjpeg), asi limitando el tipo de archivo a solo imagenes.
		- se comprueba que el tamaño de la imagen sea inferior a 2 Mb.
 
 *validar_mail:
 		- se valida que el valor que se le pasa sea distinto a false, se utiliza para verificar que mail (autor o museo) se guardará en la base de datos para ser mostrado en el front end.

Metodos: 
  Protegidos:
  	*OpenCloseForm:
 		- crea el tag de apertura y cierre del formulario.
 		- dentro del tag se incluye la variable data, la cual contiene los campos del formulario.
 		- se le asigna un action, id y clase por los atributos.
 		
	*Fieldset:
 		- crea el tag de apertura y cierre del fieldset.
 		- dentro del tag se incluye la variable data, la cual contiene los campos del fieldset.
 		- se le asigna una clase por el atributo.

 		- crea el tag de un input.
 		- dentro del tag se incluye la variable data, la cual ingresa los datos para el type, id, name, value y otros opcionales o relativos al type del input (checked, class, etc).

 		- crea el tag de un textarea.
 		- dentro del tag se incluye los datos para el id, name, rows, cols y value del textarea.
 		
 	*Select:
 		- crea el tag de un select.
 		- dentro del tag se incluye los datos para el id y name. Ademas se inluye la variable options, que contiene las opciones del select.

 	*Options:
 		- crea el tag de un option.
 		- dentro del tag se incluye los datos para el value y la texto de la option. 	

 	*Label:
 		- crea el tag de un label.
 		- dentro del tag se incluye los datos para el for y el texto del label.

		*creacionError:
			Esta clase se encarga de crear un log de errores para almacenar los errores que ocurren cuando se generan consultas incompletas, o en el caso de un archivo faltante. La clase crea el error en un archivo txt con el resultado de la consulta a la base de datos, el query enviado, el cual pudo generar problemas, ademas de asignarle la fecha y hora del error, ademas se verifica que el directorio donde se va a guardar el log de errores se haya creado correctamente, y verifica que el archivo del log se haya creado y escrito con exito, ademas de que este se haya cerrado despues de su escritura.
